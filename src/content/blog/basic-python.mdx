## Simplicity is Sophistication

Python is loved for its readability. It's the multi-tool of the programming world, used for web development (Django/Flask/FastAPI), data science, machine learning, automation, scripting, and systems tooling. Its design philosophy — "there should be one obvious way to do it" — makes it both beginner-friendly and powerful enough to run the world's most advanced AI research.

### 1. Indentation

Python uses indentation instead of curly braces to define code blocks. This forces clean, readable code as a language-level requirement — not a style guide suggestion.

```python
# Correct — consistent 4-space indentation:
def greet(name):
    if name:
        print(f"Hello, {name}!")
    else:
        print("Hello, stranger!")

# ❌ Wrong — mixing indentation causes IndentationError:
def broken():
  x = 1
    y = 2   # IndentationError: unexpected indent
```

#### Variables & Assignment

Python is dynamically typed — no type declarations needed. Variables are created on assignment.

```python
name     = "Nokib"       # str
age      = 25            # int

is_dev   = True          # bool
nothing  = None          # NoneType (equivalent of null)

# Multiple assignment:
x, y, z  = 1, 2, 3
a = b = c = 0

# Type checking:

print(isinstance(age, int))  # True

# Type conversion:
int("42")        # 42
float("3.14")    # 3.14
str(100)         # "100"
bool(0)          # False — falsy values: 0, "", [], {}, None
```

#### Lists — ordered, mutable sequences

```python
fruits = ["apple", "banana", "cherry", "date"]

# Indexing & slicing:
fruits[0]          # "apple"   — first item
fruits[-1]         # "date"    — last item
fruits[1:3]        # ["banana", "cherry"]  — slice
fruits[::-1]       # reversed list

# Common methods:
fruits.append("elderberry")       # add to end
fruits.insert(1, "avocado")       # insert at index
fruits.remove("banana")           # remove by value
fruits.pop()                      # remove & return last item
fruits.pop(0)                     # remove & return item at index
fruits.sort()                     # sort in-place (alphabetical)
fruits.sort(reverse=True)         # sort descending
fruits.reverse()                  # reverse in-place
len(fruits)                       # length
"apple" in fruits                 # True — membership check

# List comprehension — concise list building:
squares   = [x**2 for x in range(1, 6)]         # [1, 4, 9, 16, 25]
even_sq   = [x**2 for x in range(10) if x % 2 == 0]  # [0, 4, 16, 36, 64]
flat      = [n for row in [[1,2],[3,4]] for n in row] # [1, 2, 3, 4]
```

#### Tuples — ordered, immutable sequences

```python

x, y        = point          # tuple unpacking

# Use tuples for data that shouldn't change:

```

#### Sets — unordered, unique values

```python
tags   = {"python", "backend", "api"}
other  = {"python", "data", "ml"}

tags.add("fastapi")
tags.discard("api")           # remove if present, no error if not

# Set operations:
tags | other                  # union:        all unique values from both
tags & other                  # intersection: only in both
tags - other                  # difference:   in tags but not in other
tags ^ other                  # symmetric difference: in one but not both

# Fast deduplication:
unique = list(set([1, 2, 2, 3, 3, 3]))  # [1, 2, 3]
```

#### Dictionaries — key-value pairs

```python
user = {"name": "Nokib", "role": "Dev", "age": 25}

# Accessing values:
user["name"]                   # "Nokib" — KeyError if missing
user.get("email")              # None — safe access, no error
user.get("email", "N/A")       # "N/A" — with default

# Modifying:
user["email"] = "nokib@dev.io" # add or update
del user["age"]                # delete key
user.pop("role")               # remove and return value

# Iterating:
for key in user:               # iterate over keys
    print(key, user[key])

for key, value in user.items():  # iterate over key-value pairs
    print(f"{key}: {value}")

# Dictionary comprehension:
squares = {x: x**2 for x in range(1, 6)}  # {1:1, 2:4, 3:9, 4:16, 5:25}

# Merging dictionaries (Python 3.9+):
defaults = {"theme": "light", "lang": "en"}
overrides = {"theme": "dark"}
merged = defaults | overrides   # {"theme": "dark", "lang": "en"}
```

### 3. Functions

```python
# Basic function:
def add(a, b):
    return a + b

# Default parameters:
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

greet("Nokib")             # "Hello, Nokib!"
greet("Nokib", "Hey")      # "Hey, Nokib!"

# *args — variable positional arguments:
def total(*numbers):
    return sum(numbers)

total(1, 2, 3, 4, 5)       # 15

# **kwargs — variable keyword arguments:
def build_profile(**data):
    return data

build_profile(name="Nokib", role="Dev", city="Dhaka")
# {"name": "Nokib", "role": "Dev", "city": "Dhaka"}

# Combined:
def mixed(required, *args, **kwargs):
    print(required, args, kwargs)

mixed("must", 1, 2, 3, x=10, y=20)
# "must" (1, 2, 3) {"x": 10, "y": 20}
```

#### Decorators

Decorators wrap a function to extend its behavior without modifying it.

```python
import time
from functools import wraps

def timer(func):
    @wraps(func)  # preserve original function metadata
    def wrapper(*args, **kwargs):
        start  = time.time()
        result = func(*args, **kwargs)
        end    = time.time()
        print(f"{func.__name__} ran in {end - start:.4f}s")
        return result
    return wrapper

@timer
def slow_task():
    time.sleep(1)
    return "done"

slow_task()  # "slow_task ran in 1.0012s"

# Decorator with arguments:
def repeat(n):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def say_hello():
    print("Hello!")

say_hello()  # prints "Hello!" 3 times
```

### 4. Object-Oriented Programming

```python
class Animal:
    # Class variable — shared across all instances:
    kingdom = "Animalia"

    def __init__(self, name: str, sound: str):
        # Instance variables:
        self.name  = name
        self.sound = sound

    def speak(self) -> str:
        return f"{self.name} says {self.sound}!"

    def __repr__(self) -> str:
        return f"Animal(name={self.name!r})"

    def __str__(self) -> str:
        return self.name


class Dog(Animal):
    def __init__(self, name: str, breed: str):
        super().__init__(name, "Woof")   # call parent __init__
        self.breed = breed

    def fetch(self, item: str) -> str:
        return f"{self.name} fetches the {item}!"

    # Override parent method:
    def speak(self) -> str:
        return f"{self.name} barks: WOOF WOOF!"


rex = Dog("Rex", "Labrador")
print(rex.speak())         # "Rex barks: WOOF WOOF!"
print(rex.fetch("ball"))   # "Rex fetches the ball!"
print(rex.kingdom)         # "Animalia" — inherited class variable
```

#### Dataclasses — clean data containers

```python

from datetime import datetime
```

### 5. Error Handling

```python
# Basic try/except/else/finally:
def divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Cannot divide by zero")
        return None
    except TypeError as e:
        print(f"Invalid types: {e}")
        return None
    else:
        print("Division successful")  # runs only if no exception
        return result
    finally:
        print("This always runs")     # cleanup code

# Multiple exceptions in one block:
try:
    data = int(input("Enter a number: "))
except (ValueError, TypeError) as e:
    print(f"Input error: {e}")

# Custom exceptions:
class InsufficientFundsError(Exception):
    def __init__(self, amount, balance):
        self.amount  = amount
        self.balance = balance
        super().__init__(f"Cannot withdraw {amount}. Balance is only {balance}.")

class BankAccount:
    def __init__(self, balance: float):
        self.balance = balance

    def withdraw(self, amount: float):
        if amount > self.balance:
            raise InsufficientFundsError(amount, self.balance)
        self.balance -= amount
        return self.balance

account = BankAccount(100)
try:
    account.withdraw(200)
except InsufficientFundsError as e:
    print(e)  # "Cannot withdraw 200. Balance is only 100."
```

### 8. Async Python

Python supports async/await for non-blocking I/O — critical for web servers and data pipelines.

```python
import asyncio
import aiohttp

# Basic coroutine:
async def fetch_user(session, user_id):
    async with session.get(f"https://api.example.com/users/{user_id}") as resp:
        return await resp.json()

# Run multiple requests concurrently:
async def fetch_all_users(user_ids: list[int]):
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_user(session, uid) for uid in user_ids]
        users = await asyncio.gather(*tasks)  # run all concurrently
        return users

# Run the async function:
users = asyncio.run(fetch_all_users([1, 2, 3, 4, 5]))
```
